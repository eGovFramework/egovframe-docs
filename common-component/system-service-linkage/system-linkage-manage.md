## 개요

사전에 정의된 `연계기관`과 `연계메시지(전문)` 구조를 바탕으로, 실제 연계 스케줄과 배치 작업을 설정하여 데이터 동기화 프로세스를 통제하는 컴포넌트입니다.

## 주요 기능

* **연계 작업 정의**: 어떤 기관과 어떤 전문 포맷을 사용하여 데이터를 주고받을지 구체적인 연계 Job(작업)을 정의합니다.
* **배치 스케줄링**: 실시간 인터페이스가 아닌 파일 연계나 DB 연계의 경우, 새벽 시간대 등에 주기적으로 데이터(예: 일일 마감 데이터)를 전송하도록 스케줄(Cron)을 설정합니다.
* **재처리(Retry) 기능**: 네트워크 장애나 데이터 오류 등으로 연계가 실패한 건에 대해 관리자가 확인 후 수동으로 재송신(재처리)을 지시할 수 있습니다.

## 데이터 구조 (테이블)

* `COMTNSYSTEMCNTC`: 연계 작업별 고유 ID, 연계 대상, 제공 방식, 스케줄 정보, 최근 실행 결과 등을 저장하는 마스터 테이블

## 활용 방안

공공데이터포털 연계, 은행권 펌뱅킹망 데이터 수신, 혹은 그룹 계열사 간 마스터 데이터 동기화 등 대규모 배치성 인터페이스를 중앙에서 스케줄링하고 통제하는 데 쓰입니다.
