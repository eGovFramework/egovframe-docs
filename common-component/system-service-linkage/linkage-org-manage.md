## 개요

시스템과 데이터를 주고받는 외부 연계 대상 기관(예: 행정안전부, 국세청, 신용평가사 등) 또는 내부 타 시스템의 정보, 담당자 연락처, 접속 권한 등을 마스터로 등록하여 관리하는 컴포넌트입니다.

## 주요 기능

* **연계 기관 정보 등록**: 데이터를 송신하거나 수신할 대상 기관의 공식 명칭, 기관 코드, 시스템명, 사용 프로토콜(HTTP, TCP/IP) 등을 등록합니다.
* **담당자 및 인증 정보 관리**: 해당 연계 시스템을 담당하는 양측 담당자의 연락처와, API 호출 시 필요한 API Key, 인증서, IP 접근 제어(ACL) 목록 등을 관리합니다.

## 데이터 구조 (테이블)

* `COMTNCNTCSYSTEM`: 연계 대상 기관(시스템) 고유 ID, 기관명, 시스템명, IP 주소, 담당자명, 연락처 등을 저장하는 마스터 테이블

## 활용 방안

대규모 공공 시스템이나 금융 시스템과 같이 수십 개의 이기종 시스템과 복잡하게 얽혀 있는 환경에서,
연계 장애 발생 시 신속하게 타겟 시스템의 담당자를 파악하고 인증 키 만료 등을 사전에 관리하기 위한 인프라 대장 역할을 수행합니다.
