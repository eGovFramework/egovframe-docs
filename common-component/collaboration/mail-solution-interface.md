## 개요

자체 시스템이 아닌 기업 내 구축된 상용 그룹웨어 메일 솔루션(Exchange, WebMail 등)이나 외부 대량 메일 발송 서비스(SMTP, API)와 연동하여 메일을 발송하는 인터페이스 컴포넌트입니다.

## 주요 기능

* **메일 발송 API 연계**: 시스템 내에서 비밀번호 찾기, 알림 발송 등 이벤트가 발생했을 때 연동된 메일 솔루션의 API(REST, SOAP 등)를 호출하여 메일을 발송합니다.
* **SMTP 발송 처리**: 솔루션 API가 없는 경우 표준 SMTP 프로토콜(JavaMail 등)을 이용하여 외부 메일 서버를 거쳐 메일을 전송합니다.
* **발송 결과 수신**: 연동된 메일 솔루션으로부터 발송 성공, 실패(수신자 거부 등) 피드백을 받아와 시스템 로그에 기록합니다.

## 데이터 구조 (테이블)

* `COMTNEMLBGNDLOG`: (또는 별도 연계 이력 테이블) 시스템에서 메일 솔루션으로 발송을 요청한 이력과 솔루션 측의 응답 코드를 저장하는 테이블

## 활용 방안

메일 서버를 직접 구축하지 않고 기존 인프라(사내 웹메일 시스템) 자원을 재활용하여 대고객/대직원 알림 서비스를 안정적으로 운영하고자 할 때 필수적인 인터페이스 모듈입니다.
