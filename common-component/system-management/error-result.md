## 개요

`에러애플리케이션` 컴포넌트를 통해 데이터베이스에 적재된 시스템 에러(Exception) 내역을 관리자가 조회하고, 해당 에러에 대한 조치 결과를 등록/관리하는 백오피스 컴포넌트입니다.

## 주요 기능

* **에러 내역 조회**: 기간별, 에러 종류별(예: NullPointerException 등) 발생 내역을 검색하고 상세 스택 트레이스를 열람합니다.
* **조치 결과 등록**: 특정 에러에 대해 개발자나 운영자가 버그 패치, 데이터 수정 등의 조치를 완료한 후 조치 일자와 조치 내용을 기록합니다.
* **처리 상태 관리**: 에러 항목을 '미조치', '조치완료' 등으로 상태 관리하여 미해결된 시스템 결함을 추적합니다.

## 데이터 구조 (테이블)

* `COMTNERRORINFO`: 적재된 에러 내역 레코드의 조치자, 조치 일시, 조치 내용, 완료 여부 상태 컬럼을 업데이트하여 활용합니다.

## 활용 방안

시스템 런칭 이후 유지보수 단계에서 발생하는 버그 현황을 모니터링하고, 개발팀과 운영팀이 시스템 안정화를
위해 결함(Defect)을 추적하고 처리 이력을 남기는 ITS(Issue Tracking System)의 경량화 버전으로 사용됩니다.
