## 개요

`배치작업관리`를 통해 실행된 주기적인 배치 프로그램(Job)들의 실제 실행 이력과 결과(성공, 실패 원인 등)를 상세하게 모니터링하는 관리자 컴포넌트입니다.

## 주요 기능

* **배치 실행 이력 조회**: 언제 배치가 시작되었고 종료되었는지(소요 시간 포함), 그리고 실행 결과 상태(COMPLETED, FAILED 등)를 조회합니다.
* **오류 상세 확인**: 배치가 실패(FAILED)한 경우, 예외 메시지(Exception)나 에러 로그 내역을 확인하여 원인을 분석할 수 있도록 제공합니다.
* **재실행 판단**: 에러 내역을 확인한 후 문제의 원인을 조치하고, 필요시 배치작업관리에서 재실행할 수 있는 근거 데이터를 제공합니다.

## 데이터 구조 (테이블)

* `COMTNBATCHRESULT`: 각 배치 실행 건에 대한 로그 정보, 시작/종료 일시, 에러 메시지, 결과 코드를 저장하는 테이블 (Spring Batch 메타 테이블과 연동되기도 함)

## 활용 방안

시스템 관리자나 유지보수 운영자가 새벽에 수행된 배치 작업이 정상적으로 완수되었는지 아침에 점검하고, 실패한 트랜잭션을 빠르게 대응하는 데 필수적인 로그 화면입니다.
