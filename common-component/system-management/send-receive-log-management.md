## 개요

자사 시스템과 외부 타 기관 시스템(또는 내부 시스템 간)에 데이터를 연계(송신 및 수신)하는 과정에서 발생하는 트랜잭션 로그를 기록하고 조회하는 컴포넌트입니다.

## 주요 기능

* **송수신 이력 기록**: 데이터 연계 요청 시간, 요청 대상(URL, IP), 송수신된 메시지(Payload)의 크기 및 고유 식별자, 처리 결과(성공/실패)를 기록합니다.
* **오류 메시지 보존**: 통신 실패나 데이터 파싱 에러 등으로 연계가 실패했을 경우, 수신한 원본 메시지와 에러 코드를 남겨 원인 분석을 돕습니다.
* **로그 검색**: 기간별, 연계 대상 시스템별 송수신 이력을 조회합니다.

## 데이터 구조 (테이블)

* `COMTNTRSMKNDLOG`: 외부 연계 송수신 발생 일시, 연계 ID, 결과 코드 등을 저장하는 테이블

## 활용 방안

API 호출, EAI, SSO 연동 등 이기종 시스템 간의 데이터 통신이 잦은 환경에서, 통신 장애가 발생했을 때 장애 지점이 우리 측인지 상대 측인지 원인을 규명하기 위한 증적 자료로 활용됩니다.
