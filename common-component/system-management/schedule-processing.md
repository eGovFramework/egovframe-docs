## 개요

특정 시간이나 일정한 주기마다 시스템 내부의 특정 서비스(Java 메서드 등)가 자동으로 실행되도록 타이머를 설정하고 관리하는 컴포넌트입니다.

## 주요 기능

* **스케줄 등록 및 설정**: 실행할 대상 객체(Bean)와 메서드명, 그리고 실행 주기(Cron 표현식)를 설정하여 스케줄러에 등록합니다.
* **실행 모니터링**: 등록된 스케줄에 의해 서비스가 정상적으로 호출되었는지, 실행 중에 오류가 발생하지 않았는지 결과를 모니터링합니다.

## 데이터 구조 (테이블)

* `COMTNSCHDULINFO`: 스케줄 처리 대상, 실행 주기, 최근 실행 일시 등의 설정 정보를 저장하는 테이블

## 활용 방안

대규모 데이터 처리나 트랜잭션을 수반하는 `배치작업관리`와 달리, 시스템 내부의 가벼운 상태 업데이트, 세션 만료 처리, 캐시 갱신 등 주기적으로 실행되어야 하는 단순 백그라운드 작업을 제어할 때 사용됩니다.
