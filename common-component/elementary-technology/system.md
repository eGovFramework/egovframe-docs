# 시스템

## 개요

시스템 그룹은 파일·디렉터리 처리(생성·복사·이동·압축·동기화·업로드 등)와 서버·프로세스·네트워크·메모리 등 시스템 자원 점검, 각종 모니터링(HTTP·DB서비스·파일시스템) 기능을 제공하는 요소기술 묶음이다. 소스는 주로 `egovframework.com.utl.sim`, `egovframework.com.utl.sys` 패키지에 위치한다.

## 관련 문서

- [DB서비스모니터링](./db-service-monitoring.md) — DB서비스모니터링은 DB 서버의 서비스 상태를 주기적으로 모니터링하는 기능을 제공한다.
- [HTTP서비스모니터링](./http-service-monitoring.md) — HTTP서비스모니터링은 웹 서비스(웹서버 및 WAS)가 정상적인지 주기적인 점검을 수행하고, 문제 발생 시 해당 정보를 관리자에게 통보하는 기능을 제공한다.
- [XML 데이터조립](./xml-data-assembly.md) — 일반 구조적 형태의 데이터를 XML 데이터로 조립하여 파일로 저장하는 기능을 제공한다.
- [XML 데이터파싱](./xml-data-parsing.md) — XML 파일을 로드한 후 XML 데이터를 파싱하여 구조적인 데이터 형태로 변환하는 기능을 제공한다.
- [네트워크상태체크](./network-status-check.md) — Ping Test를 통해 네트워크 통신가능 여부를 확인하는 기능을 제공한다.
- [네트워크서비스모니터링](./network-service-monitoring.md) — **네트워크서비스모니터링**은 네트워크서비스 상태를 주기적으로 모니터링하는 기능을 제공한다.
- [네트워크정보확인](./network-info-check.md) — 네트워크 정보인 IP ADDRESS, MAC ADDRESS, PortScan 리스트를 확인하는 기능을 제공한다.
- [디렉토리 압축/해제](./directory-compress-extract.md) — 비즈니스 로직을 처리하면서 필요한 파일 또는 디렉토리의 압축 및 압축 해제를 위한 공통 기능을 제공한다.
- [디렉토리감시](./directory-watch.md) — 특정 디렉토리를 대상으로 디렉토리 내의 시스템에 존재하는 파일과 디렉토리의 변동 내역을 모니터링한다.
- [디렉토리권한체크](./directory-permission-check.md) — 비즈니스 로직을 처리하면서 필요한 디렉토리의 접근 권한을 확인하기 위한 공통 기능을 제공한다.
- [디렉토리복사](./directory-copy.md) — 비즈니스 로직을 처리하면서 필요시 디렉토리의 전체 구조와 파일을 복사하는 공통 기능을 제공한다.
- [디렉토리삭제](./directory-delete.md) — 비즈니스 로직을 처리하면서 필요 시 특정 디렉토리를 삭제할 수 있는 공통 기능을 제공한다.
- [디렉토리생성](./directory-create.md) — 디렉토리생성(Directory Create) 기능은 공통컴포넌트의 요소기술 중 하나로, 비즈니스 로직을 처리하면서 필요에 따라 시스템상에 지정된 경로로 디렉토리를 생성할 수 있는 공통 기능을 제공한다.
- [디렉토리속성정보체크](./directory-attribute-check.md) — 디렉토리 속성정보인 디렉토리명, 생성일자(최종수정일자), 소유계정, 읽기권한, 쓰기권한, 접근권한, 사이즈 정보를 확인하는 기능을 제공한다.
- [디렉토리이동](./directory-move.md) — 비즈니스 로직을 처리하면서 필요시 디렉토리를 이동할 수 있는 공통 기능을 제공한다.
- [디렉토리일자체크](./directory-date-check.md) — 비즈니스 로직을 처리하면서 필요 시 디렉토리의 일자 정보를 확인하기 위해 제공되는 공통컴포넌트 요소기술 중 하나이다.
- [디렉토리존재체크](./directory-exist-check.md) — 비즈니스 로직을 처리하면서 필요시 디렉토리의 존재를 확인할 수 있는 공통 기능을 제공한다.
- [디스크속성정보체크](./disk-attribute-check.md) — 특정 파일이 위치한 디스크의 마운트 경로(속성 정보)를 확인하는 기능을 제공한다.
- [디스크유효용량체크](./disk-capacity-check.md) — 시스템 내 디스크의 전체 용량, 사용 가능한 용량, 사용률 및 마운트 위치 정보를 확인하는 기능을 제공한다.
- [디스크존재체크](./disk-exist-check.md) — 시스템에 존재하는 디스크(DISK), CD, USB 등에 대한 존재를 확인하는 기능이다.
- [로그인세션정보체크](./login-session-check.md) — 로그인세션정보체크는 로그인한 세션정보를 체크하여 정의한 페이지로 이동하게 하는 기능을 제공한다.
- [로그패턴검사](./log-pattern-check.md) — 로그패턴검사(로그패턴분석)는 로그파일을 분석하여 특정 문자열 등 패턴과 일치하는 내용을 추출하는 기능을 제공한다.
- [서버자원모니터링](./server-resource-monitoring.md) — 서버자원모니터링은 시스템의 서버 자원(CPU, Memory 등) 상태를 모니터링하고 그 결과를 로그로 등록하여 관리하는 공통컴포넌트이다.
- [서버정보확인](./server-info-check.md) — 시스템에 설치된 다양한 서버(웹서버, WAS서버, 메일서버)의 정보를 확인하는 기능을 제공한다.
- [송수신모니터링](./send-receive-monitoring.md) — 송수신모니터링은 연계기관과의 송수신 상태를 주기적으로 모니터링하는 기능을 제공한다.
- [시스템정보확인](./system-info-check.md) — 시스템 고유의 속성정보인 프로세서 ID, 호스트, 운영체제(OS), 메모리(Memory), 디스크(HDD) 정보를 확인하는 기능을 제공한다.
- [웹표준검사](./web-standard-check.md) — 웹표준검사는 웹 브라우저 호환성을 위해 웹표준을 준수했는지를 검사하는 기능을 제공한다.
- [유효메모리체크](./valid-memory-check.md) — 시스템에서 사용 가능한 유효메모리를 체크하는 기능을 제공한다.
- [클라이언트정보확인](./client-info-check.md) — 리퀘스트(HttpServletRequest)를 요청한 클라이언트의 웹 브라우저 버전, 클라이언트 IP 주소를 리턴하는 기능을 제공한다.
- [파일권한체크](./file-permission-check.md) — 비즈니스 로직을 처리하면서 필요한 파일의 접근 권한을 확인하기 위한 공통 기능을 제공한다.
- [파일다운로드](./file-download.md) — 서버에 저장된 파일을 사용자가 웹 브라우저를 통해 다운로드할 수 있는 기능을 제공한다.
- [파일동기화](./file-sync.md) — 파일동기화는 파일 저장 및 삭제에 대하여 여러 AP 서버에 동기화시키는 기능으로, AP 서버들 간 동기화를 통해 파일 첨부 및 다운로드 기능을 제공한다.
- [파일변환](./file-conversion.md) — 문서 파일(Doc, XLS, PPT 등)을 PDF 형식으로 변환하는 기능을 제공한다.
- [파일보안](./file-security.md) — 암호화 알고리즘을 사용하여 파일을 암호화하거나 역으로 복호화하는 기능을 제공한다.
- [파일복사](./file-copy.md) — 지정한 경로의 파일을 다른 경로로 복사하는 기능을 제공한다.
- [파일비교](./file-compare.md) — 파일비교(File Compare) 기능은 공통컴포넌트의 요소기술 중 하나로, 서버 및 클라이언트 응용 애플리케이션에서 두 파일의 정보를 비교하는 기능을 제공한다.
- [파일삭제](./file-delete.md) — 비즈니스 로직을 처리하면서 필요시 파일을 삭제할 수 있는 공통 기능을 제공한다.
- [파일생성](./file-create.md) — 지정한 경로에 새로운 텍스트 파일을 생성하는 기능을 제공한다.
- [파일속성정보체크](./file-attribute-check.md) — 파일 속성정보인 파일명, 최종수정일자, 소유계정, 읽기권한, 쓰기권한, 접근권한, 마운트 경로 정보를 확인하는 기능을 제공한다.
- [파일송/수신](./file-send-receive.md) — FTP 프로토콜을 사용하여 파일을 송수신하는 기능을 제공한다.
- [파일시스템모니터링](./file-system-monitoring.md) — 파일시스템모니터링은 파일시스템 상태를 주기적으로 모니터링하는 기능을 제공한다.
- [파일압축/해제](./file-compress-extract.md) — 비즈니스 로직을 처리하면서 필요한 파일 또는 디렉토리의 압축 및 압축 해제를 위한 공통 기능을 제공한다.
- [파일업로드](./file-upload.md) — 첨부파일에 대한 업로드 기능을 제공한다. `MultipartHttpServletRequest` 및 `MultipartFile`을 통해 첨부된 파일 정보를 취득하고 서버에 저장하는 기능을 처리한다.
- [파일이동](./file-move.md) — 비즈니스 로직을 처리하면서 필요시 파일을 이동할 수 있는 공통 기능을 제공한다.
- [파일일자체크](./file-date-check.md) — 파일의 최종 수정일자를 확인하거나, 수정일자 조건별로 파일 목록을 조회하는 기능을 제공한다.
- [파일존재체크](./file-exist-check.md) — 디렉토리에 파일이 존재하는지 체크하는 기능을 제공한다.
- [파일파싱](./file-parsing.md) — 사용자가 업로드한 파일 및 서버(Server)의 텍스트 파일을 특정 구분자(`,`, `|`, `TAB`)로 파싱하여 정보를 추출하는 기능을 제공한다.
- [프로세스ID확인](./process-id-check.md) — 시스템에서 동작하는 프로세스에 대한 정보를 확인하는 기능을 제공한다. 본 기능은 전자정부 표준프레임워크 공통컴포넌트 요소기술 내에 구성되어 있다.
- [프로세스모니터링](./process-monitoring.md) — 프로세스모니터링은 특정 프로세스가 기동 중인지 주기적으로 점검을 수행하고, 문제 발생 시 해당 정보를 관리자에게 통보하는 기능을 제공한다.
- [프로퍼티](./property.md) — 애플리케이션에서 공통으로 사용할 프로퍼티 정보를 로드하여 파싱 후 메모리에 가지고 있는 기능을 제공한다.
- [프록시서비스](./proxy-service.md) — **프록시서비스**는 네트워크 연결에 대한 프록시 설정을 동적으로 관리하고, 연결에 대한 기본 서비스(telnet, ftp, jdbc 연결)에 대한 로깅을 기록할 수 있는 네트워크 서비스 기능을 제공한다.
